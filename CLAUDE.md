# OneBit Style Roguelike 개발 지침서

본 문서는 TUI(Terminal User Interface) 기반 턴제 로그라이크 게임 개발을 위한 클로드 코드(Claude Code)의 핵심 행동 강령입니다. 모든 개발 작업은 이 지침을 최우선으로 준수해야 합니다.

## 🛠️ 기술 스택 및 환경
* **언어:** ISO C99 (Standard C)
* **빌드 시스템:** GNU Make (gcc)
* **출력 방식:** ASCII 기반 TUI (표준 입출력 활용)

## 📋 핵심 명령어
* **빌드:** `make all`
* **정리:** `make clean`
* **테스트:** `make test`

## 📏 아키텍처 및 코딩 원칙
1. **관심사 분리 (SoC):** 게임 로직(`src/logic`)과 화면 렌더링(`src/renderer`)을 완전히 분리합니다. 로직 코드 안에는 `printf` 등 화면 출력 함수가 절대 들어가서는 안 됩니다.
2. **에러 핸들링:** 모든 함수의 반환값은 성공/실패를 나타내는 상태 코드(`int`)여야 하며, 실제 결과값은 포인터 인자를 통해 전달받습니다. 예외 발생 시 `goto ERROR_EXIT;` 레이블을 통해 자원을 일괄 해제합니다.
3. **스타일 가이드 준수:** `docs/style.md`에 명시된 명명 규칙(포인터 `p_`, 전역 `g_` 등)을 엄격히 따릅니다.
4. **TDD (테스트 주도 개발):** 새로운 기능을 짤 때는 `tests/` 폴더에 실패하는 테스트 케이스를 먼저 작성한 후, 이를 통과시키는 최소한의 코드를 구현합니다.

## 🔄 워크플로우 (Claude Code 행동 지침)
1. 작업을 시작하기 전 반드시 `plan.md`를 읽고 현재 진행해야 할 단계를 파악합니다.
2. 코드를 생성하기 전 `docs/architecture.md`를 참고하여 설계 의도에 맞는지 확인합니다.
3. 작업이 완료되면 `make test`로 회귀 테스트를 수행하고, `plan.md`의 체크박스(`[ ]` -> `[x]`)를 업데이트하여 보고합니다.
